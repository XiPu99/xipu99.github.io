How to create a Hugo website

27 Dec 2019

Install Hugo

Follow the instructions on Hugo's website to install Hugo.

All your posts are in markdown format and you can edit them under ./content/posts directory.

Once you are satisfied with how your blog looks, you can then call:

hugo -D

This will build all the static pages and the output will be in ./public/ directory.

Deployment

Now you just need to deploy these static pages. To do that, you need to have a domain. I used the domain provided by GitHub. To get that, you just need to create a new GitHub repo with a name in the format “youraccountname.github.io”. More directions can be found here.

After you get a domain name, you need to change placeholder base url baseURL = "https://example.org/" in the config.toml file by setting it to be your own domain name.

Once you create the repo, you can just upload your ./public/ directory to GitHub and access your blog with the url https://youraccountname.github.io/.
